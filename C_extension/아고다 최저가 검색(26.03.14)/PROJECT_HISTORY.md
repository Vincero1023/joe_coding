# Agoda Best 개발 히스토리

## 개요

이 문서는 최초 아이디어부터 현재 `Agoda Best 2.0` 상태까지, 어떤 문제를 겪었고 어떻게 고쳤는지를 시간 순서대로 기록합니다.

## 1. 출발점

사용자 목표는 단순했습니다.

- Agoda 호텔 상세 페이지를 열고
- 확장 프로그램을 실행해서
- 국가/유입/카드 링크에 따라 더 싼 가격이 있는지 자동으로 찾기

초기 가설은 `userCountry`, `currency`, `language`, `cid/tag`에 따라 가격이 달라질 수 있다는 것이었습니다.

## 2. 초기 MVP

처음에는 현재 Agoda 호텔 URL을 바탕으로 몇 가지 프리셋을 순차 비교하는 구조로 시작했습니다.

초기 구현 특징:

- 현재 탭 URL 기준 비교
- 백그라운드 탭 생성 후 순차 스캔
- 팝업에서 실행
- 최저가 링크로 이동

하지만 곧 아래 문제가 드러났습니다.

- 잘못된 숫자를 가격으로 인식
- 현재 페이지 기준가를 잘못 읽음
- 국가/IP 영향은 브라우저만으로 완전 재현 불가

## 3. 가격 추출 오류와 수정

가장 많은 시간이 들어간 영역은 가격 추출이었습니다.

실제 발생한 문제:

- `11 KRW`처럼 작은 잡숫자를 가격으로 오인
- `최저 월 9,204 할부 결제`를 객실가로 오인
- `숙소(1,868)` 같은 숙소 개수를 가격으로 오인
- 기준가와 비교가의 차액 계산이 뒤틀림
- 항공 위젯의 `ICN - PEK 시작가`를 호텔 가격으로 오인

주요 수정:

- 할부/월결제 텍스트 필터링
- 숙소 개수, 리뷰 수, 평점 제외
- `시작가`와 `1박` 중심의 사용자 규칙 반영
- 항공 관련 문맥 제외
- `ICN` 같은 공항 코드를 통화로 인식하지 않도록 whitelist 적용

## 4. 결과 UI 진화

초기에는 팝업 안에서 결과를 보려 했지만, 사용 흐름이 불안정했습니다.

문제:

- 팝업을 닫으면 사용자가 맥락을 잃음
- 스캔 중 다른 탭으로 가면 진행 상황을 놓치기 쉬움
- 링크 열기와 결과 확인이 섞여 불편함

개선:

- 별도 결과 탭 도입
- 스캔 중에도 결과 탭이 유지됨
- 상위 결과와 전체 결과를 고정 화면에서 확인 가능
- 필요한 링크를 언제든 열 수 있음

## 5. 중지 기능과 회귀

사용자가 페이지를 바꾸면 이전 페이지 결과가 이어지는 문제가 생겨 중지 기능을 추가했습니다.

이 과정에서 한 차례 회귀가 발생했습니다.

- DOM 준비 타이밍이 너무 빨라져 가격을 읽기 전에 추출 시도
- `Could not extract a visible price` 반복

이후 로딩 대기와 가격 추출 시점을 다시 안정화했습니다.

## 6. 링크 카탈로그 확장

단순 프리셋 비교에서 벗어나, 사용자가 수집한 실제 Agoda 링크 패턴을 카탈로그로 정리했습니다.

분류:

- 검색 경로
- 카드/결제
- 항공사/적립
- 프로모션 페이지

이 단계에서 `cid`, `tag`, 일반 프로모션 랜딩 페이지를 분리해 다루게 됐습니다.

## 7. 단계별 스냅샷과 롤백

변경이 커질수록 회귀 위험이 높아져, 각 단계마다 스냅샷을 남기는 구조를 추가했습니다.

도입 파일:

- `tools/start-step.ps1`
- `tools/rollback.ps1`
- `STEP_LOG.md`
- `snapshots/`

이후 중요한 변경은 모두 롤백 가능한 단계로 저장됐습니다.

## 8. 버전 1.x 안정화

- 결과 탭 기반 흐름 안정화

### 1.2

- 시스템 설명 문서화
- 버전 표기 정리

### 1.3

- 병렬 스캔 구조 도입 후 안정화
- 동시성 `3`으로 조정
- 가격 앵커를 `시작가`, `1박` 기준으로 고정

### 1.4

- 새로 발견한 실험용 CID 추가

### 1.5

- 우선순위 스캔
- `desktop/mobile` 변형 비교
- 숨김 탭 풀 유지

## 9. 다른 컴퓨터에서만 실패한 문제

동일한 코드라도 다른 컴퓨터에서 실패하는 사례가 있었습니다.

주요 원인:

- Agoda가 가격 속성을 먼저 `0`으로 렌더링
- fallback 경로에서 정규식 처리 버그
- 통화 코드 오인
- DOM 구조/렌더링 타이밍 차이

핵심 수정:

- `0` 속성값은 버리고 텍스트 fallback 강화
- `matchAll` fallback 버그 수정
- 통화 whitelist 적용
- 항공 위젯 제외

이 수정 이후 기기별 편차가 크게 줄었습니다.

## 10. 예약 추적기 도입

최저가 찾기에서 끝나지 않고, 이미 예약한 호텔 가격을 계속 추적하고 싶다는 요구가 추가됐습니다.

도입 기능:

- 결과 탭에서 현재 예약 저장
- Agoda URL 직접 붙여 넣기 등록
- 예약 목록 페이지
- 수동 재확인
- 가격 하락 감지
- 24시간 자동 점검
- 가격 하락 알림

이 단계에서 확장 프로그램은 단순 스캐너에서 예약 추적 도구로 확장됐습니다.

## 11. 브랜드 정리: Agoda Best 2.0

마지막으로 제품 이름과 화면 구성을 정리했습니다.

변경 사항:

- 이름을 `Agoda Best`로 통일
- 팝업/결과 탭/예약 추적기 제목 통일
- 아이콘과 파비콘 새 디자인 적용
- 레이아웃 정리
- 문서 전체 정리

## 12. 현재 상태

현재 버전은 `2.0.0`입니다.

핵심 상태:

- 스캔 동시성 `3`
- 숨김 스캔 탭 재사용
- 카드/검색/항공사/프로모션 비교
- `desktop/mobile` 변형 비교
- 결과 탭 유지
- 예약 추적기 포함
- 단계별 롤백 가능

## 13. 앞으로 확장 가능한 영역

- 실패 원인 진단 모드 강화
- 환경별 DOM 패턴 자동 수집
- 국가/IP 라우팅 기반 고급 비교
- 쿠폰/회원가 별도 모드
- 예약 추적 요약 대시보드
