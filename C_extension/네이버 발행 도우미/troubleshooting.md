## 발행 자동화 트러블슈팅 로그

반복적으로 겪은 성공/실패 패턴과 환경 팁을 정리했습니다. 새 PC/브라우저에서 문제가 생기면 아래를 먼저 확인하세요.

### 공통 환경 팁
- 크롬: 게스트/새 프로필도 가능하지만 확장 허용·스토리지 권한이 필요. 광고 차단 등으로 리소스가 막히면 발행 버튼 탐색 실패 가능.
- 페이지 로드 완료 후 실행: 에디터/헤더가 늦게 뜨면 버튼을 못 찾을 수 있으니 1~2초 기다린 뒤 단축키/팝업 실행.
- 단축키: `Alt+Shift+P`를 캡처 단계에서 막도록 수정(Chrome 탭 그룹 진입 방지). 그래도 브라우저가 가로채면 팝업 버튼으로 실행.

### 기능별 성공/실패 사례
- **발행 버튼 찾기**
  - 성공: `.btn_publish`, `aria-label/title`에 `발행` 포함, 텍스트에 `발행/예약/등록/출간/publish` 포함된 버튼/링크/submit 입력값이 노출될 때.
  - 실패: 로딩 지연으로 DOM 미노출(6회×0.7초 재시도 후에도 없음), 가시성이 `display:none/visibility:hidden` 또는 광고차단·정책으로 버튼이 숨겨진 경우.
  - 조치: 페이지 완전히 열린 뒤 실행, 광고차단 해제, 에디터 상단 영역이 보이는지 확인.
- **단축키 오동작(탭 그룹 등으로 이동)**
  - 실패: `Alt+Shift+P`가 Chrome 기본 동작으로 먼저 처리될 때.
  - 조치: 현재 캡처 리스너로 기본 동작 차단함. 여전히 발생 시 확장 팝업 버튼으로 실행하거나 크롬 다른 단축키와 겹치는지 확인.
- **날짜/시간 설정**
  - 성공: “예약” 라벨에서 Tab 1회로 날짜 버튼 진입 → ArrowLeft+Space×4로 달력 초기화 → (필요 시) Tab+Space로 월 이동 → Tab 횟수로 날짜 선택 → 시간/분 입력.
  - 실패: 달력이 열리기 전에 키 입력, 달력 DOM이 포커스를 잃은 상태, 시간/분 입력 필드가 비가시/비활성인 경우.
  - 조치: 발행 예약 영역이 화면에 보이는지 확인, 팝업/레이어가 가려져 있지 않은지 확인.
- **카테고리 선택**
  - 성공: 드롭다운 버튼 노출 후 옵션 텍스트가 설정값과 일치/포함.
  - 실패: 설정값 미입력, 다른 계정/블로그에서 카테고리명이 다른 경우.
  - 조치: 설정 값 확인, 다른 블로그에서는 카테고리 값을 맞춰 저장.

### 현재 증상 기록
- 새 PC/브라우저에서 `Alt+Shift+P` 사용 시 Chrome 그룹탭 화면으로 진입 → 캡처 리스너로 차단하도록 수정.
- 같은 환경에서 “발행 버튼을 찾지 못했습니다” 경고 발생 → 버튼 노출 전 실행 가능성 높음. 상단 UI가 보이는지 확인 후 재시도 필요.

### 추가 확인이 필요한 경우
- 발행 버튼의 실제 DOM 정보(aria-label/title/class/data-*)를 콘솔로 확인해 알려주면 셀렉터에 추가 가능.
- 페이지가 iframe에 감싸져 있다면 상단 프레임(`window.top.document`) 접근이 막히는지 확인 필요.
- 20251128 새 환경에서 발행 버튼 미탐지 -> 제공된 DOM (.publish_btn__m9KHH 등) 기반으로 스크립트 단순화 및 재시도 로직 추가. 단축키는 캡처 단계에서 Chrome 기본 동작 차단.
