# AI Rules (강제)

## 절대 금지
- 기존 기능 삭제 금지.
- 기존 파일/폴더 이동 금지.
- 요청 없는 대규모 리팩터링 금지.

## 변경 방식
- 변경은 작은 단위로 나누고, 각 단계가 실행 가능해야 한다.
- 한 번에 하나의 목적만 수정하고 즉시 검증한다.
- 실패 시 원인과 롤백 포인트를 먼저 기록한다.

## 출력 방식
- 항상 수정 파일 경로를 명시한다.
- 각 파일에서 수정 위치(함수명/라인 근처/섹션명)를 명시한다.
- 실행한 검증 명령과 결과(성공/실패)를 짧게 남긴다.

## Windows 우선 규칙
- 경로는 Windows 기준(`C:\...`, `\\server\share\...`)으로 해석한다.
- 인코딩은 기본 UTF-8, 호환 이슈가 있을 때만 UTF-8-SIG 사용 여부를 명시한다.
- 줄바꿈은 CRLF 기준으로 처리하고, 혼합 줄바꿈을 만들지 않는다.

## 프로젝트 타입별 규칙
- windows: 배치파일/포터블 실행을 우선 고려하고 더블클릭 실행 경로를 제공한다.
- web-docker: `docker-compose.yml` 기준으로 구성하고 `.env` 분리 및 포트 매핑을 명시한다.
- chrome-extension: Manifest V3 기준을 지키고 권한은 최소 범위만 선언한다.
- scripts: 단일 파일 또는 최소 파일 수, 간단 CLI 인자 구조를 우선한다.

## 안전
- 네트워크 드라이브 작업 시 `Path.resolve()`와 절대경로를 먼저 확인해 원본 덮어쓰기를 방지한다.
